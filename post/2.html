关键字

1、bool

  • C语言中没有原生的bool类型的关键字，平时看到的也是typedef int bool

  • C++中有原生的bool关键字

  • ~~~c++ void func(int, int) void func(bool, bool) 在C++中就是函数重载，int和bool是不一样的

3、逻辑运算

  • and => &&
  • or => ||
  • not => !

4、位运算

  • bitand &
  • bitor |
  • xor ^
  • and_eq &=
  • or_eq |=
  • xor_eq ^=
  • compl ~
  • not_eq !=

5、引用

  • ~~~c++ int a = 1; int &b = a; int c = 22; b = c; //这个地方只是把22赋值给了b而已，不是改变了绑定 要注意const这个关键字

7、Union

  • 共用体，测试大小端的，需要掌握
  • 在C++中，定义的时候，不用写Union了，没有什么区别

8、inline

  • 是一个用于实现的关键字，不是声明的关键字，就是要和函数体放在一起，不是和声明放在一起
  • 递归不能被inline，因为inline是编译的时候原地展开，而递归在编译阶段还不知道调用了几次，所以不可以
  • 有循环或者是复杂的判断的函数不建议被inline
  • class类中，如果函数的定义在类里面，就默认是inline了，感觉说了也是白说

9、nullptr

  • nullptr是C++中正经空指针，感觉也没有什么用

10、assert

  • assert是用于运行阶段的，如果出现了错误，程序就G了，还是不建议，还不如用if呢

  • assert的作用是先计算表达式 expression ，如果其值为假（即为0），那么它先向stderr打印一条出错信息，然后通过调用 abort 来终止程序运行

  • static_assert静态断言好用一点，是编译阶段的

  • ~~~~c++ static_assert(2==2, "error!");//因为2等于2，所以不会报错，不然就G了

13、cast

  • ~~~c++ //static_cast:源生类型之间的隐式类型转换，可以用static_cast来明确告知编译器，避免警告，转换后可能丢失精度，正确性需要程序员自己保证 int a = 9; char w = 's'; int b; char c; a = static_cast(w);

  • ~~~C++ //const_cast:用来修改类型的const或volatile属性,一般用在函数传参的时候，但是我感觉不写好像也没有什么关系，或者是用C中的强制类型转换也行 const int a = 2; int &b = const_cast<int&>(a);

14、auto

  • ~~~C++ auto x = 1;//自动推导x是int，可以typeid试试看 const int i = 2; auto y = i;//这个地方y是不会保留const的

  • decltype是保留了const，就是可以推测出const类型，但是auto不会推测出const

16、static

  • 在C++中还有两种用法就是修饰静态成员函数和静态成员变量，修饰了之后就代表这个成员是这个类的了，不是对象的

  • ~~~c++ class A { public: static int a ; static void func(void) }; int A::a = 2;

19、mutable

  • ~~~c++ class A { public: mutable int a = 2;//用了mutable修饰之后，a变量就可以在func中被修改了 int b = 2; int func(int, int) const;//代表这个函数不会修改类的成员变量就是a、b }; int A::func(int a, int b) const { return a; }
